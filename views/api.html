Authentification et requêtes

Requetes HTTP

Afin d'executer une requête via l'API Snapcall, vous devrez passer l'api_key et l'api_secret en même temps que le reste de vos paramètres.
Toutes les requêtes envoyées à l'API Snapcall doivent être de type HTTP POST.
Le résultat de vos requêtes vous sera renvoyé au format JSON.

Les requêtes vers l'API peuvent avoir la forme suivante:

curl [-d "<parameter>=<value>"]{0,} <URI>

La réponse de l'API se presentera sous cette forme:

[index_key: {
 key_1: "value",
 key_2: "value",
 ...
}]

Environnement

Pour gagner du temps, sauvegardez vos clefs directement dans l'environnement et utilisez les exemples pour tester notre API.

$> export SNAPI_KEY={{api_key}}

$> export SNAPI_SECRET={{api_secret}}

Connection

Se connecter sur un profil utilisateur.

POST http://api.snapcall.io/user/login

Paramètres d'entrée

Nom Type Mandatoire Description Exemple
email Email Oui Email de l'utilisateur hello@snapcall.io
password Password Oui Mot de passe de l'utilisateur snapcall1234

Retourne NULL si aucun utilisateur n'est trouvé

Créer un nouveau compte Snapcall

Nom Type Mandatoire Description Exemple

Nom Type Mandatoire Description Exemple

schedule Array Non Les horaires d'ouverture de votre widget (pour chaque jour de la semaine). 09:00-19:30;09:00-19:30;09:00-19:30;09:00-19:30;09:00-19:30;00:00-00:00;00:00-00:00

msg_widget_ongoing string Non Message d'appel en cours Appel en cours ...

opt_browser bool Non En cas d'incompatibilité du navigateur, 'Y' pour afficher un message d'erreur, 'N' pour masquer le widget Y

opt_secure char Non Si le site web est non sécurisée (non https), 'Y' pour rediriger vers une page hébergée sécurisée, 'N' pour masquer le widget Y
opt_accessibility bool Non Notice d'accessibilité, 'Y' pour l'activer, 'N' pour la désactiver. Y
opt_log bool Non

Paramètres de retour

Renvoie TRUE si l'opération a réussie, FALSE sinon.
